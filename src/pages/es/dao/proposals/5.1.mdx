## Resumen

El equipo de ENS labs ha estado trabajando en una nueva versión del oráculo DNSSEC y el registrador DNS que, combinado con resolución wildcard (ENSIP 10) y CCIP-Read, permite 'DNSSEC sin gas' - habilitando el uso de nombres DNS dentro de ENS sin transacciones en la cadena requeridas. Esta propuesta reemplaza el registrador DNSSEC existente con el nuevo.

Los nombres DNS existentes continuarán funcionando como antes, y los nombres aún pueden ser importados usando el método 'legacy'. El nuevo registrador también permite configurar un nombre estableciendo un registro TXT en \_ens.name.tld, conteniendo la dirección de un contrato resolver especial a usar, seguido por cualquier dato específico del resolver. Los resolvers diseñados para el propósito pueden ser configurados para analizar estos datos extra, haciendo la configuración completamente fuera de la cadena una posibilidad.

Junto con los nuevos contratos registrador y oráculo, se proporciona un resolver simple que lee la dirección Ethereum para resolver un nombre desde los datos extra.

Post-ejecución, ENS Labs ejecutará un proceso para actualizar todos los TLDs DNS actuales para usar el nuevo registrador. Los TLDs solo funcionarán con el nuevo registrador una vez que esta transacción (sin permisos) sea enviada para el TLD.

## Especificación

Llamar `setController` en el contrato Root en `root.ens.eth`, pasando la dirección del nuevo registrador DNS (`0xb32cb5677a7c971689228ec835800432b339ba2b`).
