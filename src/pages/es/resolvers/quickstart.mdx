# Inicio Rápido de Resolvers

En el corazón de cada nombre ENS está su resolver. Un resolver es un contrato inteligente que implementa un conjunto específico de características de Resolver (ver [Interfaz de Resolver](/es/resolvers/interfaces)).
Las funciones del contrato inteligente resolvers tienen control sobre el proceso de resolución de un ["nodo"](/es/resolution/names#namehash) (un nombre o subdominio) y en adelante (subdominios de sí mismo).

## Resolver Básico

Un ejemplo ingenuo pero muy plausible de un resolver es el siguiente.

```solidity
contract MyResolver {
    function addr(bytes32 node) external pure returns (address) {
        return 0x225f137127d9067788314bc7fcc1f36746a3c3B5;
    }

    function supportsInterface(
        bytes4 interfaceID
    ) external pure returns (bool) {
        return
            interfaceID == this.addr.selector ||
            interfaceID == this.supportsInterface.selector;
    }
}
```

Observa cómo el anterior siempre devolvería la misma dirección independientemente del nombre por el que se consulte.

Si quieres [escribir tu propio resolver](/es/resolvers/writing), o ver la [referencia de interfaz](/es/resolvers/interfaces).

## Resolver Público

El resolver por defecto para todos los nombres es el Resolver Público, una navaja suiza de resolvers, escrito por el equipo de ENS Labs, lleno de características con todo lo que tu usuario cotidiano podría necesitar.
Puedes leer más sobre el [Resolver Público](/es/resolvers/public).

## Interactuando con un resolver

Dependiendo del resolver a cargo de un nombre, ciertas aplicaciones frontend también podrán interactuar con ellos.
Esto significa que puedes establecer tus registros favoritos, actualizar tu nombre a diferente lógica, y más, desde tu dApp de elección.

¿Estás escribiendo una dApp y quieres construir esto? Revisa la sección [Interactuando con un Resolver](/es/resolvers/interacting).

## Resolución Fuera de la Cadena

Aunque por defecto la resolución ENS se hace en la cadena. Puedes aprovechar el poder de CCIP Read para redirigir la resolución a una puerta de enlace fuera de la cadena.
Más sobre escribir un resolver habilitado para CCIP Read [aquí](/es/resolvers/ccip-read).
